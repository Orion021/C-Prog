Introduction to C

History, characteristics, when to choose C, toolchain setup, and simple complete examples.

What is C?

Definition

C is a general-purpose, procedural programming language developed in the early 1970s by Dennis Ritchie at Bell Labs. It is a low-level language compared to modern high-level languages — offering direct access to memory through pointers, predictable performance, and a small standard library. C has influenced many later languages (C++, C#, Java, Rust, Go) and remains widely used for system software, embedded systems, and performance-critical applications.

Key characteristics / features

  • Procedural programming model (functions, modular code).
  • Low-level memory access using pointers.
  • Small runtime — minimal standard library.
  • Portable across platforms with standards (ANSI C / ISO C).
  • Deterministic performance; suitable for systems programming and embedded devices.
  • Compilation model: source (.c) → preprocess → compile → assemble → link → executable.

When to choose C

Choose C when you need fine-grained control over memory, minimal runtime overhead, or are writing code for OS kernels, device drivers, embedded microcontrollers, or performance-critical libraries. For application-level rapid development or heavy standard libraries (GUI, web), higher-level languages may be more productive.

Toolchain & compilation

Typical toolchain

  • Compiler: gcc (GNU), clang (LLVM), or vendor compilers.
  • Build tools: make, cmake, or simple shell scripts.
  • Editor/IDE: VS Code, Vim, Emacs, CLion, Dev-C++ (Windows), etc.

Basic compilation commands:

// compile and link in one step
gcc hello.c -o hello

// compile only (produce object file hello.o)
gcc -c hello.c -o hello.o

// compile with warnings and C standard (recommended)
gcc -std=c11 -Wall -Wextra -pedantic hello.c -o hello

Hello World — complete example

A minimal but complete C program

#include <stdio.h>

int main(void) {
    // print a string to standard output
    printf("Hello, world!\n");
    return 0; // indicate success
}

Save as hello.c, then compile and run:

gcc -std=c11 -Wall -Wextra hello.c -o hello
./hello

Example: Simple calculator (command-line)

Demonstrates input, basic operators, and error checking

#include <stdio.h>
#include <stdlib.h>

int main(void) {
    double a, b;
    char op;

    printf("Enter an expression (e.g. 2.5 * 4): ");
    if (scanf("%lf %c %lf", &a, &op, &b) != 3) {
        fprintf(stderr, "Invalid input\n");
        return 1;
    }

    switch (op) {
        case '+': printf("%g\n", a + b); break;
        case '-': printf("%g\n", a - b); break;
        case '*': printf("%g\n", a * b); break;
        case '/':
            if (b == 0) { fprintf(stderr, "Division by zero\n"); return 1; }
            printf("%g\n", a / b);
            break;
        default:
            fprintf(stderr, "Unknown operator: %c\n", op);
            return 1;
    }

    return 0;
}

Note: using scanf is fine for small examples, but for robust input handling prefer reading a line with fgets and parsing with sscanf or strtod.

Beginner best-practices

  • Compile with warnings enabled (-Wall -Wextra) and fix warnings early.
  • Initialize variables before use; uninitialized variables cause undefined behaviour.
  • Check results of I/O and memory allocation functions (malloc, fopen, etc.).
  • Prefer size_t for sizes/indices and int for small integers unless platform-specific widths are required.
  • Free dynamically allocated memory and avoid memory leaks.
