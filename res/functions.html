Functions

Definition, syntax, complete example programs and outputs, things to remember, and guidance on when and how to use functions in C.

Definition

A function is a named block of code that performs a specific task. Functions help organize code into reusable, testable units and improve readability and maintainability.

Syntax & Types

Basic function parts:

  • Return type — type of value the function returns (use void for none).
  • Function name — identifier.
  • Parameter list — types and names of input values (can be empty).
  • Function body — statements enclosed in braces.
// general form
return_type function_name(type1 param1, type2 param2) {
    /* body */
    return value; // if return_type is not void
}

Function declarations (prototypes) tell the compiler about a function's signature before use:

int add(int a, int b); // prototype

int main(void) {
    int s = add(2,3);
}

int add(int a, int b) { return a + b; }

Parameter passing: Call by value

C passes function arguments by value. The function receives copies of the arguments — modifications inside the function do not affect the caller's variables unless pointers are used.

// pass_by_value.c
#include <stdio.h>

void inc(int x) {
    x = x + 1; // modifies local copy
}

int main(void) {
    int a = 5;
    inc(a);
    printf("a = %d\n", a); // still 5
    return 0;
}

Simulating pass-by-reference using pointers

To modify caller variables, pass pointers (addresses) and dereference inside the function.

// pass_by_pointer.c
#include <stdio.h>

void inc(int *p) {
    (*p) = (*p) + 1; // modifies caller's variable
}

int main(void) {
    int a = 5;
    inc(&a);
    printf("a = %d\n", a); // now 6
    return 0;
}

Recursion

A function may call itself. Useful for problems naturally defined recursively (factorial, tree traversal). Ensure a base case to prevent infinite recursion.

// recursion_factorial.c
#include <stdio.h>

int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

int main(void) {
    printf("5! = %d\n", factorial(5));
    return 0;
}

Output: 

5! = 120

Storage classes & Related Keywords

  • static — for functions, restricts visibility to the translation unit (file). For variables, preserves value across calls.
  • extern — used to declare a function or variable defined in another file (default for functions).
  • inline — a hint to the compiler to inline function body (may or may not be inlined).
// static function example
static int helper(int x) { return x * 2; } // only visible in this .c file

Function Pointers

Functions have addresses and can be pointed to by function pointers — useful for callbacks and tables of functions.

// function_pointer.c
#include <stdio.h>

int add(int a,int b){return a+b;}
int mul(int a,int b){return a*b;}

int main(void){
    int (*op)(int,int) = add; // pointer to function
    printf("add: %d\n", op(2,3));
    op = mul;
    printf("mul: %d\n", op(2,3));
    return 0;
}

Complete Example: Modular program with functions

Program: simple student record operations split into functions.

// students.c
#include <stdio.h>

void print_header(void) {
    printf("ID  Name        Marks\n");
    printf("--------------------\n");
}

void print_student(int id, const char *name, int marks) {
    printf("%2d  %-10s %4d\n", id, name, marks);
}

int main(void) {
    print_header();
    print_student(1, "Alice", 82);
    print_student(2, "Bob", 74);
    return 0;
}

Output: 

ID  Name        Marks
--------------------
 1  Alice        82
 2  Bob          74

Things to remember

  • Declare function prototypes before use or define functions before callers to avoid implicit declaration warnings.
  • Functions receive copies of arguments — use pointers to modify caller variables.
  • Match function signature (types and number of parameters) precisely.
  • Return meaningful values. Use void for no return value.
  • Avoid deep recursion that may overflow the call stack; convert to iterative solutions if needed.
  • Use static to limit function visibility to a file (encapsulation in C).

When to use functions

  • Break large problems into smaller, focused tasks.
  • When a piece of logic is reused in multiple places.
  • To isolate complex logic for testing and maintenance.
  • To create a clear program structure: API-like separation between modules.
