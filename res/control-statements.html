Control Statements

Definition, syntax, example programs with output, things to remember, and guidance on when to use each control statement in C.

Definition

Control statements direct the flow of execution in a program. They allow branching (conditional execution), repetition (loops), and jumping (break/continue/goto).

if statement

Syntax:

if (condition) {
    /* statements */
}

Example program:

// if_example.c
#include <stdio.h>
int main(void) {
    int n = 5;
    if (n > 0) {
        printf("n is positive\n");
    }
    return 0;
}

Output:

n is positive

When to use: Simple conditional execution when you only need to run code for the true case.

Remember: condition is considered true if non-zero and false if zero.

if-else statement

Syntax:

if (condition) {
    /* true block */
} else {
    /* false block */
}

Example program:

// if_else_example.c
#include <stdio.h>
int main(void) {
    int x = 10;
    if (x % 2 == 0) {
        printf("x is even\n");
    } else {
        printf("x is odd\n");
    }
    return 0;
}

Output:

x is even

When to use: Use when you have two alternative paths.

Nested if / if-else if

Syntax:

if (cond1) {
    // ...
} else if (cond2) {
    // ...
} else {
    // ...
}

Example:

// nested_if_example.c
#include <stdio.h>
int main(void) {
    int score = 78;
    if (score >= 90) printf("A\n");
    else if (score >= 75) printf("B\n");
    else if (score >= 60) printf("C\n");
    else printf("F\n");
    return 0;
}

Output:

B

When to use: Multiple exclusive conditions; keep order from most specific to least.

switch-case statement

Syntax:

switch (expression) {
  case constant1: stmts; break;
  case constant2: stmts; break;
  default: stmts;
}

Example program:

// switch_example.c
#include <stdio.h>
int main(void) {
    int day = 3;
    switch (day) {
        case 1: printf("Mon\n"); break;
        case 2: printf("Tue\n"); break;
        case 3: printf("Wed\n"); break;
        default: printf("Other\n");
    }
    return 0;
}

Output:

Wed

When to use: When you need to choose between many discrete constant values (integers or enum). Use break to avoid fall-through. Fall-through is allowed intentionally for grouped cases.

switch only works with integral types (char, int, enum). Expressions must be constant at compile time for case labels.

for loop

Syntax:

for (initialization; condition; update) {
    /* body */
}

Example program:

// for_example.c
#include <stdio.h>
int main(void) {
    for (int i = 0; i < 5; i++) {
        printf("i = %d\n", i);
    }
    return 0;
}

Output:

i = 0
i = 1
i = 2
i = 3
i = 4

When to use: Best when you know the iteration count ahead of time or can express loop progress with an index.

while loop

Syntax:

while (condition) {
    /* body */
}

Example program:

// while_example.c
#include <stdio.h>
int main(void) {
    int i = 0;
    while (i < 5) {
        printf("i = %d\n", i);
        i++;
    }
    return 0;
}

Output:

i = 0
i = 1
i = 2
i = 3
i = 4

When to use: Use when the number of iterations is not known in advance and depends on a condition.

do-while loop

Syntax:

do {
    /* body */
} while (condition);

Example program:

// do_while_example.c
#include <stdio.h>
int main(void) {
    int i = 0;
    do {
        printf("i = %d\n", i);
        i++;
    } while (i < 3);
    return 0;
}

Output:

i = 0
i = 1
i = 2

When to use: Use when the loop body must execute at least once regardless of the condition.

break, continue, and goto

break — exits the nearest loop or switch immediately.

// break_example.c
#include <stdio.h>
int main(void) {
    for (int i = 0; i < 10; i++) {
        if (i == 3) break; // loop stops when i == 3
        printf("%d ", i);
    }
    printf("\nDone\n");
    return 0;
}

Output:

0 1 2 
Done

continue — skip the rest of the current loop iteration and proceed with next iteration.

// continue_example.c
#include <stdio.h>
int main(void) {
    for (int i = 0; i < 5; i++) {
        if (i == 2) continue; // skip printing 2
        printf("%d ", i);
    }
    printf("\nDone\n");
    return 0;
}

Output:

0 1 3 4 
Done

goto — jumps to a labeled statement. Use sparingly; can make code hard to follow.

// goto_example.c
#include <stdio.h>
int main(void) {
    int n = 0;
    start:
    if (n < 3) {
        printf("n = %d\n", n);
        n++;
        goto start; // jumps back to label
    }
    return 0;
}

Output:

n = 0
n = 1
n = 2

Prefer structured control flow (loops & functions). Use goto mainly for error handling in C where it can improve cleanup code readability.

Things to remember

  • Conditions evaluate to non-zero (true) or zero (false).
  • Be careful with operator precedence in conditions — add parentheses when in doubt.
  • Always ensure loop termination to avoid infinite loops (unless intentional).
  • Use break and continue to control loop flow, but avoid overusing them as they can reduce readability.
  • Prefer switch for multiple discrete alternatives instead of chained if-else when cases are constant.
  • Avoid modifying the loop variable in the loop header and body in conflicting ways.
  • When using scanf inside loops, clear input buffer appropriately to avoid unexpected behavior.
