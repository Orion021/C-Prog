Structure of a C Program

Understand how a C program is organized: headers, functions, main(), and comments. Learn how the compiler processes it.

Basic building blocks

  • Documentation Section — Comments explaining what the program does.
  • Link Section — Header files inclusion (#include <stdio.h>).
  • Definition Section — Symbolic constants using #define or global variables.
  • Global Declaration Section — Function declarations and global variables.
  • main() Function Section — Starting point of program execution.
  • Subprogram Section — User-defined functions.

Typical structure example

/* Documentation section */
// Program: Sum of Two Numbers
// Author: Krishna

/* Link section */
#include <stdio.h>

/* Definition section */
#define PI 3.14159

/* Global declaration section */
int sum(int a, int b); // function declaration

/* main() function section */
int main(void) {
    int x = 5, y = 7, result;
    result = sum(x, y);
    printf("Sum = %d\n", result);
    return 0;
}

/* Subprogram section */
int sum(int a, int b) {
    return a + b;
}

Explanation of each part

  • Comments: Non-executable lines used to document code (// for single-line, /* ... */ for multi-line).
  • #include: Inserts content of a header file. <stdio.h> provides input/output functions like printf() and scanf().
  • #define: Used to define constants or macros.
  • Global variables: Declared outside functions, accessible throughout the program.
  • Function declarations: Tell the compiler about a function’s name, return type, and parameters before its use.
  • main() function: Entry point of every C program; returns an integer status to the OS.
  • User-defined functions: Modularize code for reusability and clarity.

Compilation flow

The C compilation process involves four major stages:

  1. Preprocessing: Handles directives like #include and #define.
  2. Compilation: Converts C code into assembly language.
  3. Assembly: Translates assembly into object code (.o).
  4. Linking: Combines object files and libraries into an executable program.
gcc -Wall -std=c11 program.c -o program
./program

Example: Multiple functions in one file

#include <stdio.h>

void greet(void) {
    printf("Hello, welcome to C programming!\n");
}

int add(int a, int b) {
    return a + b;
}

int main(void) {
    greet();
    printf("Sum = %d\n", add(3, 4));
    return 0;
}

Tips

  • Always start with main() and build additional functions gradually.
  • Use comments to explain purpose and logic, not obvious details.
  • Group related functions in separate source files for large programs.
  • Enable warnings with -Wall -Wextra to catch syntax issues early.
